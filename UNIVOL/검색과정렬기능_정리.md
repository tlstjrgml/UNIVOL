# 봉사게시판 검색과 정렬 기능 구현 정리

## 📋 전체 흐름

```
사용자 입력 (검색어, 정렬방식, 페이지)
    ↓
HTML Form/Button (post.html)
    ↓
PostController.selectAll() 메서드
    ↓
PostService / PostMapper (DB 조회)
    ↓
결과를 Model에 담기
    ↓
post.html에서 결과 출력
```

---

## 1️⃣ HTML 층 - 사용자 입력 (post.html)

### 검색 폼 & 정렬 드롭다운
```html
<form class="filter-group" method="get" action="/post">
  <!-- 정렬 방식 선택 -->
  <select id="sort-field" name="sort" onchange="applyFilter()">
    <option value="latest" th:selected="${sort == 'latest'}">최신순</option>
    <option value="views_desc" th:selected="${sort == 'views_desc'}">조회수 높은순</option>
    <option value="views_asc" th:selected="${sort == 'views_asc'}">조회수 낮은순</option>
  </select>
  
  <!-- 검색어 입력 -->
  <input th:value="${keyword}" type="text" name="keyword" 
         id="search-input" class="search-input" 
         placeholder="제목 또는 작성자 검색" />
  
  <!-- 검색 버튼 -->
  <button type="submit" class="search-btn" onclick="applyFilter()">검색</button>
</form>
```

**파라미터 설명:**
- `name="sort"` → sort 파라미터로 Controller에 전달
- `name="keyword"` → keyword 파라미터로 Controller에 전달
- `th:selected="${sort == 'latest'}"` → 전에 선택한 정렬방식 유지
- `th:value="${keyword}"` → 전에 입력한 검색어 유지

---

### 페이지네이션 버튼
```html
<div class="pagination">
  <th:block th:each="n:${#numbers.sequence(pi.startPage, pi.endPage)}">
    <!-- 숨겨진 form (화면에 안 보임) -->
    <form th:id="'pageForm-' + ${n}" action="/post" method="GET" style="display:none;">
      <input type="hidden" name="page" th:value="${n}">
      <input type="hidden" name="keyword" th:value="${keyword}">
      <input type="hidden" name="sort" th:value="${sort}">
    </form>
    
    <!-- 페이지 버튼 -->
    <button type="button" th:text="${n}"
            th:onclick="'goToPage(' + ${n} + ')'"
            class="page-btn"
            th:classappend="${n == pi.currentPage} ? 'active' : ''">
    </button>
  </th:block>
</div>

<script>
  function goToPage(pageNum) {
    const form = document.getElementById('pageForm-' + pageNum);
    form.submit();
  }
</script>
```

**왜 이렇게 하나?**
- Thymeleaf 3.1+는 onClick에 문자열 변수를 직접 못 넣음 (보안)
- 대신 form을 이용해서 파라미터 전달
- `goToPage()` 함수는 해당 페이지의 form을 submit

## 2️⃣ 컨트롤러 층 - 요청 처리 (PostController.java)

```java
@GetMapping("/post")
public String selectAll(Model model, 
        @RequestParam(value = "page", defaultValue="1") int currentPage,
        @RequestParam(value = "sort", defaultValue="latest") String sort,
        @RequestParam(value="keyword", defaultValue="") String keyword) {
    
    // 1단계: 검색 결과 개수 구하기
    int listCount;
    if(keyword.isEmpty()) {
        listCount = pService.getListCount();  // 검색 없음 → 전체 게시물
    } else {
        listCount = pService.getSearchCount(keyword);  // 검색 있음 → 검색 결과만
    }
    
    // 2단계: 페이지 정보 계산 (10개씩 나누기)
    PageInfo pi = Pagenation.getPageInfo(currentPage, listCount, 10);
    
    // 3단계: DB에서 조회할 행의 범위 계산
    int startRow = (pi.getCurrentPage()-1)*pi.getBoardLimit()+1;
    int endRow = pi.getCurrentPage() * pi.getBoardLimit();
    
    // 4단계: 게시물 조회
    ArrayList<Post> plist;
    if(keyword.isEmpty()) {
        plist = pService.selectAll(startRow, endRow, sort);  // 전체 목록
    } else {
        plist = pService.searchPosts(keyword, sort, startRow, endRow);  // 검색 결과
    }
    
    // 5단계: View에 데이터 전달
    model.addAttribute("pi", pi);           // 페이지 정보 (1,2,3,4,5 등)
    model.addAttribute("plist", plist);     // 게시물 목록
    model.addAttribute("keyword", keyword); // 검색어 (View에서 유지)
    model.addAttribute("sort", sort);       // 정렬방식 (View에서 유지)
    
    return "post/post";
}
```

**각 파라미터 설명:**
- `page` → 현재 페이지 번호 (기본값: 1)
- `sort` → 정렬 방식 (기본값: "latest" = 최신순)
- `keyword` → 검색어 (기본값: "" = 빈 문자열)

**중요한 부분:**
```
❌ 잘못된 순서:
PageInfo pi = Pagenation.getPageInfo(currentPage, pService.getListCount(), 10);  // 항상 전체 개수
int listCount = pService.getSearchCount(keyword);  // 나중에 구함

✅ 올바른 순서:
int listCount = pService.getSearchCount(keyword);  // 먼저 정확한 개수 구하기
PageInfo pi = Pagenation.getPageInfo(currentPage, listCount, 10);  // 그 개수로 페이지 계산
```

## 3️⃣ 서비스 층 (PostService.java)

```java
// 전체 게시물 개수
public int getListCount() {
    return pMapper.getListCount();
}

// 검색 결과 게시물 개수
public int getSearchCount(String keyword) {
    return pMapper.getSearchCount(keyword);
}

// 전체 게시물 조회 (정렬 포함)
public ArrayList<Post> selectAll(int startRow, int endRow, String sort) {
    return pMapper.selectAll(startRow, endRow, sort);
}

// 검색 결과 조회 (정렬 포함)
public ArrayList<Post> searchPosts(String keyword, String sort, int startRow, int endRow) {
    return pMapper.searchPosts(keyword, sort, startRow, endRow);
}
```

**역할:** Controller와 Mapper 사이의 중간 계층 (비즈니스 로직 처리)

## 4️⃣ 매퍼 층 (PostMapper.xml)

### 전체 게시물 개수
```xml
<select id="getListCount" resultType="int">
    SELECT COUNT(*) FROM post WHERE pType = 'V'
</select>
```

### 검색 결과 개수
```xml
<select id="getSearchCount" parameterType="String" resultType="int">
    SELECT COUNT(*) FROM post 
    WHERE pType = 'V' 
    AND (pTitle LIKE CONCAT('%', #{keyword}, '%') 
         OR userName LIKE CONCAT('%', #{keyword}, '%'))
</select>
```

### 전체 게시물 조회 (정렬)
```xml
<select id="selectAll" resultType="com.univol.post.model.vo.Post">
    SELECT * FROM post 
    WHERE pType = 'V'
    <choose>
        <when test="sort == 'latest'">
            ORDER BY pDate DESC
        </when>
        <when test="sort == 'views_desc'">
            ORDER BY views DESC
        </when>
        <when test="sort == 'views_asc'">
            ORDER BY views ASC
        </when>
    </choose>
    LIMIT #{startRow}, #{endRow}
</select>
```

### 검색 결과 조회 (정렬)
```xml
<select id="searchPosts" resultType="com.univol.post.model.vo.Post">
    SELECT * FROM post 
    WHERE pType = 'V'
    AND (pTitle LIKE CONCAT('%', #{keyword}, '%') 
         OR userName LIKE CONCAT('%', #{keyword}, '%'))
    <choose>
        <when test="sort == 'latest'">
            ORDER BY pDate DESC
        </when>
        <when test="sort == 'views_desc'">
            ORDER BY views DESC
        </when>
        <when test="sort == 'views_asc'">
            ORDER BY views ASC
        </when>
    </choose>
    LIMIT #{startRow}, #{endRow}
</select>
```

**SQL 설명:**
- `LIKE CONCAT('%', #{keyword}, '%')` → 제목이나 작성자 검색
- `<choose>` → 정렬 방식에 따라 다른 SQL 실행
- `LIMIT #{startRow}, #{endRow}` → 10개씩만 가져오기

## 📊 데이터 흐름 예시

### 상황: "봉사" 검색, 최신순, 페이지 2

**1. HTML에서 URL 생성**
```
/post?keyword=봉사&sort=latest&page=2
```

**2. Controller에서 받은 파라미터**
```
keyword = "봉사"
sort = "latest"
currentPage = 2
```

**3. Service/Mapper에서 SQL 실행**
```sql
-- 먼저 개수 구하기
SELECT COUNT(*) FROM post 
WHERE pType = 'V' 
AND (pTitle LIKE '%봉사%' OR userName LIKE '%봉사%')
-- 결과: 25개

-- 페이지 2의 데이터 가져오기 (11~20번째)
SELECT * FROM post 
WHERE pType = 'V'
AND (pTitle LIKE '%봉사%' OR userName LIKE '%봉사%')
ORDER BY pDate DESC
LIMIT 11, 10  -- startRow=11, endRow=20
```

**4. HTML에서 출력**
```
- 게시물 11~20번 출력
- 페이지 버튼 1~3 표시 (25개 ÷ 10 = 3페이지)
- 페이지 2 버튼만 초록색
- 검색어 "봉사" 입력창에 유지
- 정렬 드롭다운에 "최신순" 유지
```

## 🔑 핵심 개념 정리

| 기능 | 파라미터 | 역할 |
|------|---------|------|
| **검색** | `keyword` | DB에서 제목/작성자와 매칭되는 게시물만 필터링 |
| **정렬** | `sort` | 최신순/조회수높은순/낮은순으로 정렬 |
| **페이지네이션** | `page` | 10개씩 묶어서 페이지 표시 |

### 검색 + 정렬 + 페이지네이션이 함께 작동하는 이유
```java
// 검색 결과가 몇 개인지 먼저 파악
int listCount = pService.getSearchCount(keyword);

// 그 개수 기준으로 페이지 계산
PageInfo pi = Pagenation.getPageInfo(currentPage, listCount, 10);

// 해당 페이지의 데이터를 정렬 방식에 따라 조회
plist = pService.searchPosts(keyword, sort, startRow, endRow);
```

## ⚠️ Thymeleaf 에러 해결 과정

### 문제: onClick에 문자열 변수 직접 사용
```html
<!-- ❌ 에러: Thymeleaf 3.1+는 보안상 이를 금지 -->
<button th:onClick="|location.href='/post?page=${n}&keyword=${keyword}&sort=${sort}'|">
```

### 원인
- Thymeleaf 3.1+는 onClick 같은 이벤트 핸들러에 문자열을 직접 넣는 것을 금지
- 이유: 사용자가 `keyword='"; alert("hack"); //'` 같은 악성 코드를 입력할 수 있기 때문

### 해결책: Form + 함수 방식
```html
<!-- 1. 숨겨진 form에 데이터 저장 -->
<form th:id="'pageForm-' + ${n}" action="/post" method="GET" style="display:none;">
  <input type="hidden" name="page" th:value="${n}">
  <input type="hidden" name="keyword" th:value="${keyword}">
  <input type="hidden" name="sort" th:value="${sort}">
</form>

<!-- 2. 함수로만 호출 (변수 직접 안 사용) -->
<button type="button" th:onclick="'goToPage(' + ${n} + ')'">

<!-- 3. JavaScript에서 form submit -->
<script>
  function goToPage(pageNum) {
    document.getElementById('pageForm-' + pageNum).submit();
  }
</script>
```

## 📝 요약

1. **사용자 입력** → post.html의 검색 폼, 정렬 드롭다운, 페이지 버튼
2. **요청 처리** → PostController에서 3개 파라미터 받기
3. **DB 조회** → 
   - 먼저 검색 결과 개수 구하기
   - 그 개수로 페이지 정보 계산
   - 해당 페이지의 데이터를 정렬해서 조회
4. **결과 표시** → post.html에서 결과 + 검색어/정렬/페이지 상태 유지

**핵심:** 검색과 정렬과 페이지네이션이 모두 URL 파라미터로 연결되어 있어서, 각 기능을 사용해도 다른 기능의 상태가 유지된다!
